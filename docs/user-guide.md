## 1. インストールと起動

### macOS

1. `Roentgen-0.1.0-arm64.dmg`（Apple Silicon）または `Roentgen-0.1.0-x64.dmg`（Intel Mac）を開く
2. Roentgen アイコンを「アプリケーション」フォルダにドラッグ
3. 初回起動時に「開発元が未確認」と表示された場合:
   - **システム設定 → プライバシーとセキュリティ** → 「このまま開く」をクリック
4. ランチパッドまたは Spotlight から **Roentgen** を起動

### Windows

1. `Roentgen-Setup-0.1.0.exe` を実行
2. インストールウィザードに従って進める
3. デスクトップのショートカットまたはスタートメニューから起動

### Linux (AppImage)

```bash
chmod +x Roentgen-0.1.0.AppImage
./Roentgen-0.1.0.AppImage
```

### 起動後の画面

起動すると **ファイルドロップゾーン**（濃いグレーの中央エリア）が表示されます。ここにDICOMファイルまたはフォルダをドラッグするか、クリックしてファイル選択ダイアログを開きます。

## 2. DICOMファイルの読込

### 方法1: ドラッグ＆ドロップ

- `.dcm` ファイル・複数ファイル・フォルダを**ドロップゾーンにドラッグ**して放す
- フォルダを投入するとサブフォルダ内も自動でスキャンします

### 方法2: ファイル選択ダイアログ

- ドロップゾーンをクリックすると OS のファイル選択ダイアログが開く
- 複数選択（Cmd+クリック / Ctrl+クリック）にも対応

### 読込中の表示

- ヘッダーに進捗バー（読込中 n/N ファイル）が表示される
- 大量ファイル読込中は**キャンセルボタン**が表示される → クリックで中断可能

### 読込結果

| 状態 | 表示 |
|---|---|
| 正常 | 左パネルにシリーズ一覧・サムネイルが表示される |
| 一部エラー | スキップ数（例: 2件スキップ）がヘッダーに表示される |
| 非DICOMファイル | 「DICOMファイルではありません」と表示されスキップ |
| 破損ファイル | 「ファイルが破損しています」と表示されスキップ |

## 3. 画像の基本操作

### ズーム・パン

| 操作 | 方法 |
|---|---|
| ズームイン | `+` キー / マウスホイール上 |
| ズームアウト | `-` キー / マウスホイール下 |
| パン（移動） | **パンモード**でクリック＆ドラッグ |
| フィット表示 | `F` キー |
| 1:1表示 | `1` キー（実ピクセル等倍） |

### 回転・反転

| 操作 | 方法 |
|---|---|
| 90°時計回り回転 | `R` キー / ツールパネルの回転ボタン |
| 水平反転 | `H` キー / ツールパネルのミラーボタン |
| 垂直反転 | `V` キー / ツールパネルのミラーボタン |
| リセット | `0` キー（全変形をリセット） |

### インバート

- `I` キー または ツールパネルの「反転」 → ネガ/ポジ切替

### フルスクリーン

- `F11` キー で全画面モード切替

## 4. WW/WC（ウィンドウ幅・レベル）調整

WW/WC は骨・軟部・肺などの見え方を調整する最重要操作です。

### マウスドラッグで調整

1. ツールパネルで **「WW/WC」モード** を選択（またはデフォルト）
2. 画像上で:
   - **左右ドラッグ** → ウィンドウ幅（WW）: コントラスト調整
   - **上下ドラッグ** → ウィンドウレベル（WC）: 明るさ調整

### リセット

- `W` キー → DICOMヘッダーデフォルト値にリセット

## 5. 馬用プリセット

ワンクリックで臨床用途に最適化された WW/WC を適用できます。

| プリセット名 | 用途 | キー |
|---|---|---|
| 骨 (Bone) | 骨格・骨折評価 | `1` |
| 軟部 (Soft Tissue) | 腱・靭帯・筋肉 | `2` |
| 蹄骨 (Pedal Bone) | 蹄骨・内部構造 | `3` |
| 舟状骨 (Navicular) | 舟状骨症候群評価 | `4` |
| 肺 (Lung) | 胸部レントゲン | `5` |
| 腹部 (Abdomen) | 腹腔内評価 | `6` |

ツールパネルの **「馬用プリセット」** セクションからボタンでも切替可能です。

> **プリセット値はあくまで出発点です。**
> 個体差・撮影条件に応じてマウスドラッグで微調整してください。

## 6. 計測ツール（距離・角度）

DICOM の Pixel Spacing 情報を使用し、**mm単位**で計測します。

### 距離計測

1. ツールパネルの **「計測」→「距離」** を選択（またはキー `M`）
2. 計測開始点をクリック
3. 計測終了点をクリック
4. 画面上に距離（mm）が表示される

### 角度計測

1. ツールパネルの **「計測」→「角度」** を選択
2. 1点目（頂点）をクリック
3. 2点目をクリック
4. 3点目をクリック
5. 画面上に角度（°）が表示される

### 計測値の削除

- 計測マーカーをクリックして選択 → `Delete` キー
- 全計測削除: ツールパネルの「計測クリア」ボタン

> **注意**: Pixel Spacing がDICOMヘッダーに含まれていない場合、計測値は「px」単位で表示されます。主治医にDICOM取得設定の確認を依頼してください。

## 7. マルチフレーム再生（CINE）

複数フレームのDICOMシリーズを動画的に再生できます。

### 再生操作

| 操作 | 方法 |
|---|---|
| 再生/一時停止 | `Space` キー / ツールパネル再生ボタン |
| 次フレーム | `→` キー |
| 前フレーム | `←` キー |
| 最初に戻る | `Home` キー |
| 最後へ | `End` キー |
| スライダー | ツールパネル下部のスライダーをドラッグ |

### シリーズ切替

左パネルのサムネイルをクリックすると別シリーズに切替えます。

## 8. Google Drive連携

Google Drive 上の DICOM ファイルを直接 Roentgen で開けます。

### 初回セットアップ

1. ヘッダー右上の **Drive アイコン** をクリック
2. 「認証情報ファイルを配置してください」と表示された場合:
   - GCP コンソールで OAuth2 クライアント ID をダウンロード
   - `credentials.json` ファイルを指示されたパスに配置
3. **「Google アカウントで認証」** をクリック
4. ブラウザに Google 認証画面が開く → アカウントを選択して許可
5. 認証完了後、Drive 上の DICOM ファイル一覧が表示される

### ファイルの読込

1. Drive ファイル一覧から読み込みたいファイルを選択（チェックボックス）
2. **「読込」** ボタンをクリック
3. ダウンロード進捗がヘッダーに表示される
4. 完了後、ローカルファイルと同様に表示される

### ログアウト

ヘッダーの Drive アイコン → **「ログアウト」** をクリック

## 9. スクリーンショット保存

計測結果・WW/WC 調整後の状態を PNG で保存できます。

1. ツールパネルの **「スクリーンショット」** ボタンをクリック（または `S` キー）
2. ファイル保存ダイアログが開く
3. 保存先・ファイル名を指定して保存

> 計測マーカー・オーバーレイテキストも画像に含まれます。所見記録にご活用ください。

## 11. よくある質問（FAQ）

### Q. DICOMファイルを開けません

**A.** 以下を確認してください:
1. ファイル拡張子が `.dcm` であること（大文字 `.DCM` も可）
2. ファイルサイズが 0 バイトでないこと
3. ファイルのアクセス権限（読取り可能か）
4. 他のアプリでファイルがロックされていないか

### Q. 画像が真っ黒（または真っ白）に表示されます

**A.** WW/WC 値が適切でない可能性があります:
1. `W` キーで WW/WC をリセット
2. または馬用プリセット（`1`〜`6` キー）を試す
3. マウスドラッグで手動調整

### Q. 計測値が「px」で表示されます

**A.** DICOM ヘッダーに Pixel Spacing 情報が含まれていません。撮影装置の設定を確認し、DICOM 出力設定で Pixel Spacing を含めるよう依頼してください。

### Q. Google Drive 認証に失敗します

**A.** 以下を確認してください:
1. `credentials.json` が正しいパスに配置されているか
2. GCP プロジェクトで「Google Drive API」が有効になっているか
3. OAuth 同意画面でメールアドレスが承認済みか
4. インターネット接続が安定しているか

### Q. アプリが重い・画像描画が遅い

**A.** 以下をお試しください:
1. 一度に読み込むファイル数を減らす（まず1シリーズから）
2. 他のアプリを閉じてメモリを確保
3. アプリを再起動

### Q. スクリーンショットが保存できません

**A.** 保存先フォルダへの書き込み権限を確認してください。デスクトップやピクチャフォルダなど、権限のある場所を選択してください。

### Q. ウィンドウを閉じて再度開くと設定がリセットされます

**A.** v0.1.0 では、ウィンドウサイズと最後の WW/WC 値は次回起動時に自動復元されます。ファイルの再読込は手動で行ってください。

## 12. クラッシュレポート（任意）

アプリの品質向上のため、**任意でクラッシュレポートの送信**をご協力いただけます。

- **送信される情報**: エラーメッセージ・スタックトレース・OS/アプリバージョン
- **送信されない情報**: 患者データ・DICOMタグ・ファイル名・個人情報
- **設定方法**: ヘッダー右上の「クラッシュレポート」トグルをオン

送信は Sentry（セキュリティ認証済みのエラー監視サービス）を経由します。いつでもオフに戻せます。

- アプリバージョン（ヘッダーのバージョン番号）
